# Code of Conduct & Ethical Agreement

## Our Pledge

We as contributors and maintainers pledge to make participation in our project and community a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Ethical Use Guidelines

Calq MCP is designed for legitimate time tracking and productivity purposes. Users agree to:

### ✅ Acceptable Use

- Track your own work time honestly and accurately
- Log time for projects you are genuinely working on
- Use memory features for legitimate work-related notes
- Respect the privacy of shared team data
- Report bugs and security issues responsibly

### ❌ Unacceptable Use

- Falsifying time entries or billing records
- Logging time for work not performed
- Accessing or modifying other users' personal data without consent
- Using the system to surveil or monitor employees without their knowledge
- Circumventing authentication or authorization controls
- Using AI-powered features to generate deceptive content

## Privacy Commitments

### Data Collection
- Calq stores data locally in `~/.calq/data.json`
- No data is sent to external servers except:
  - Voyage AI (for memory embeddings) - content of memories
  - GitHub (for OAuth) - authentication only

### User Consent
- Team members must consent to being tracked
- Personal memories are visible only to their creator
- Shared data should be clearly marked as such

### Data Retention
- Users can delete their entries and memories at any time
- Data is not backed up to external services automatically

## For Organizations

If deploying Calq for a team, we recommend:

1. **Inform employees** that time tracking is in use
2. **Obtain consent** before enabling team features
3. **Establish clear policies** on what constitutes billable time
4. **Respect privacy** - avoid using Calq as a surveillance tool
5. **Allow corrections** - give users the ability to edit their entries

## Enforcement

Violations of this code of conduct may result in:

- Warning from maintainers
- Temporary or permanent ban from the project
- Reporting to relevant authorities (in cases of fraud or illegal activity)

## Reporting

Report issues to the maintainers via:
- GitHub Issues (for public concerns)
- Direct message to maintainers (for sensitive matters)
